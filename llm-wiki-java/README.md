## P7 vs P8 核心差距

```
P7 = 能解释原理 + 能设计常见系统场景
P8 = 有大规模生产经验 + 能做系统性权衡 + 能向上影响技术决策
```

### 差距1：工业级实践认知（vs 教科书方案）

P7 知道"Redis 预扣 + MQ 异步落盘"；P8 知道阿里的库存系统实际怎么做、小红书用 MySQL 抗秒杀的真实方案，以及两者相比教科书答案的差异在哪里。

**缺口**：`大厂实践/`（3个文件）未摄入。

### 差距2：项目影响力表达

P7 能说清技术方案；P8 能说清"我遇到了什么问题 → 做了什么决策 → 产生了多少可量化的 impact"。

**缺口**：`项目难点&亮点/`（22个文件）未摄入，缺 `synthesis/设计-项目难点表达.md`。

### 差距3：三高体系联动思维

P7 分类掌握高并发/高可用/高性能；P8 能说清三者如何相互制约：

```
高并发 → 引入异步/分片 → 牺牲强一致（最终一致）
高可用 → 引入副本/冗余 → 牺牲延迟（性能下降）
高性能 → 引入缓存/本地计算 → 牺牲一致性（脏数据风险）
```

**缺口**：`高并发/`（6文件）+ `高可用/`（8文件）+ `高性能/`（11文件）未摄入，缺 `主题-三高体系.md`。

### 差距4：分布式深水区

wiki 已有 CAP/BASE/2PC/TCC/Seata，但缺：
- 3PC（为什么实际不用）
- 拜占庭将军问题（区块链/BFT 共识场景）
- 分布式数据库选型（TiDB vs OceanBase vs 分库分表真实 trade-off）
- Leaf 算号器（百度开源，工业级 ID 生成）

**缺口**：`分布式/` 深水区文件未摄入。

### 差距5：DDD 落地（不只是概念）

wiki 有 `概念-DDD.md`（充血模型/聚合根/限界上下文），但缺实战 synthesis：
- 如何用限界上下文指导微服务拆分决策
- 聚合根粒度权衡（太细 → 跨聚合事务；太粗 → 锁竞争）

**缺口**：缺 `synthesis/设计-DDD落地实战.md`。

### 差距6：面试高频边界问题

P8 面试官会追问 P7 边界之外的问题，55个真实面经包含这些模式。

**缺口**：`面经实战/`（55个文件）未摄入。
