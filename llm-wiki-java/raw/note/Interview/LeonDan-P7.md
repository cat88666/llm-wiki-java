**① 高并发库存扣减引擎（从 0 到 1，独立设计）**

问题：电商大促场景下，多渠道并发扣减导致超卖风险与 OMS 调用阻塞，接口 P99 长达 800ms。

方案与成果：
- **原子性保障**：Redis + Lua 脚本确保并发扣减原子性，**单机 QPS 5000+，RT <5ms**
- **异步解耦**：设计 OMS 调用与 Redis 扣减分离的异步处理机制，CompletableFuture 并行调用多渠道 OMS，混单场景 **P99 从 800ms 降至 200ms**
- **策略隔离**：策略模式封装 B2B / B2C / SC 差异化扣减逻辑，各渠道独立演进，无耦合

**② 跨系统最终一致性保障**

- 设计 **Pub/Sub 事务消息 + 本地消息表 + 定时对账**三层防护机制，订单系统与库存系统最终一致性达 **99.99%**
- 所有扣减操作异步写入 MongoDB 操作日志（来源 / 数量 / 时间 / 操作者），支撑审计与异常溯源

**③ 爆款热 Key 治理**

- 大促爆款商品并发击穿单个 Redis Key，设计 **Caffeine 本地缓存 + Redis 读写分离 + 分段存储**组合方案
- 热 key 场景 **RT 降低 50%**，彻底消除单点热点访问瓶颈

**④ 实时可靠价格计算引擎（从 0 到 1，独立设计）**

问题：价格规则依赖人工发版，上线周期 2 天；多 SKU 价格生效时间不精准，缓存一致性难以保障。
