### 深圳迈克尔斯科技有限公司　2020/03 — 2023/02　｜　高级 Java 工程师

**Michaels B2C / B2B / SC 跨境电商平台**（全程核心研发）

平台规模：日均 500w+ UV，大促峰值 QPS 18,000+，SKU 数量 800w+，覆盖 B2C/B2B/SC 三个业务域，服务全球 1,200w+ 注册用户。

---

**① 高并发库存系统重构（主导）**

- **背景**：大促期间库存超卖率 0.3%，MySQL 行锁导致库存扣减 P99 延迟 > 3 s，TPS 瓶颈在 200 以内，每次大促均触发超卖投诉。
- **方案**：
  - 将库存扣减从 `SELECT FOR UPDATE` 迁移至 **Redis + Lua 原子脚本**，Lua 保证"检查库存–扣减–返回"的原子性，彻底消除 TOCTOU 竞态；
  - 扣减成功后异步发 MQ 事件落库到 OMS，实现存储层读写分离；
  - 消费端幂等设计：MQ 消息携带 `orderId` 唯一键，MySQL 唯一索引兜底，保证重复消费不重复扣库存；
  - 引入 Sentinel 对库存服务入口做限流（单 SKU QPS 阈值），超限熔断返回"库存不足"而非等待超时。
- **结果**：超卖率降为 **0**，库存扣减 P99 < 12 ms，系统 TPS 从 200 提升至 **8,500+（提升 40×）**，连续 3 次大促零超卖事故。

---

**② 商品价格系统重构（负责）**

- **背景**：全量价格同步耗时 4 h+，批量轮询导致促销价生效存在 2~30 min 滞后窗口，价格不一致投诉频发。
- **方案**：
  - 引入 **GCP Pub/Sub** 替换批量轮询，价格变更即时发布事件；消费端用 `messageId` 去重，保证幂等处理；
  - **Cloud Task** 实现延迟任务：定时促销价（如黑五 0 点开启）精确到秒级生效，替代原来的 cron 轮询；
  - **Google Spanner** 承接跨区域多写场景（美国 / 加拿大双区域），替换 MySQL 解决跨区域强一致读问题；
  - Redis 缓存热点价格（命中率 98.5%），采用 **先更新 DB → 再删除缓存** 策略，配合 TTL 30 s 兜底，避免脏缓存。
- **结果**：价格同步延迟从 4 h 降至 **P99 < 3 s**，数据不一致投诉归零，促销活动上线时间从提前 30 min 准备缩短至 5 min。

---

**③ 全链路可观测性体系建设（主导）**

- **Prometheus + Grafana**：覆盖 JVM / 服务 / 业务三层，配置 P99 延迟、错误率、Redis 内存水位、库存告警等 **35 个核心监控指标**，建立大促红线看板；
- **ELK 集中日志**：推动全团队日志结构化（JSON 格式统一），全链路 `traceId` 贯通（HTTP Header → MDC → 日志 → ES），问题排查时间从平均 **2 h 压缩到 < 15 min**；
- **SkyWalking 链路追踪**：接入 24 个微服务，自动采集慢调用（> 500 ms），每周输出 TOP-10 性能瓶颈报告，推动 3 次关键路径优化；
- **XXL-JOB 分布式调度**：替换单点 Quartz，承接 18 个核心定时任务（对账 / 库存盘点 / 价格同步），配置执行超时告警和幂等重试策略。
