# 01-简历

Hadoop集群：

Hadoop集群是一个由多个节点组成的分布式计算环境，用于存储和处理大规模数据。它包括两个核心组件：HDFS和YARN。Hadoop集群通过将数据和计算分布在多个节点上，实现了数据的并行处理和高可靠性存储。

HDFS（Hadoop分布式文件系统）：

HDFS是Hadoop集群中的分布式文件系统，用于存储大规模数据。它将数据划分为多个块，并在集群的多个节点上进行分布式存储。HDFS提供了高可靠性、容错性和扩展性，适用于大数据的持久化存储。

YARN（Yet Another Resource Negotiator）：

YARN是Hadoop集群中的资源管理系统，负责集群资源的管理和作业调度。YARN将集群的计算资源划分为多个容器（containers），并根据作业的需求进行资源分配和调度，以实现高效的并行计算。

CDH和HDP

CDH（Cloudera Distribution for Hadoop）和HDP（Hortonworks Data Platform）：都是Hadoop的发行版本，由Cloudera和Hortonworks提供。它们都是基于开源的Hadoop项目构建而成，提供了一套集成的Hadoop生态系统，包括HDFS、YARN、Hive、HBase等组件，并提供了易于管理和部署的工具。

Spark和Flink：

Spark和Flink都是大数据处理的开源框架，用于高效地进行数据处理和分析。它们提供了内存计算、流式处理和批处理等功能，并支持复杂的数据处理任务。Spark主要基于RDD（弹性分布式数据集）模型，而Flink则采用了基于流的数据流模型。两者都具有高性能和易用性，可根据具体场景选择适合的框架进行数据处理。

ODS-DW-APP层数仓分层是指在数据仓库（Data Warehouse）架构中，按照一定的规范和层次划分不同的数据存储和处理层级，以支持数据的管理、处理和应用。

通过将数据仓库划分为ODS-DW-APP三个层级，可以实现数据的分层管理和处理，使得数据的流动和使用更加灵活和高效。ODS层用于接收原始数据，DW层用于数据的清洗和整合，APP层用于数据的应用和分析，从而支持企业的决策和业务需求。

ODS层：

ODS（Operational Data Store）层是数据仓库架构中的操作型数据存储层，用于接收和存储源系统的原始数据。在ODS层，数据一般以原始的、不加工的形式存储，以保留数据的完整性和可追溯性。ODS层的数据通常用于实时或近实时的操作型需求，例如实时报表、实时监控等。

DW层：

DW（Data Warehouse）层是数据仓库架构中的数据存储和加工层，用于进行数据清洗、整合和加工，以满足分析和决策支持的需求。在DW层，数据经过清洗、抽取、转换和加载等过程，被组织成符合业务需求的结构化数据集。DW层的数据一般以主题为中心，支持复杂的分析和查询操作。

APP层：

APP（Application）层是数据仓库架构中的应用层，用于支持各类具体的业务应用和数据消费者。在APP层，基于DW层的数据，可以进行各种数据应用开发，包括报表生成、数据可视化、数据挖掘、机器学习等。APP层提供了面向业务用户的数据访问接口和工具，以便他们能够方便地使用和分析数据。

Airflow和Doris分别是两个不同的数据工具。

Airflow和Doris可以结合使用，以构建一个完整的数据流程管理和分析系统。Airflow用于任务调度、依赖管理和工作流定义，而Doris用于高性能的数据存储和查询。通过Airflow和Doris的结合，您可以实现灵活的数据流程调度和数据分析能力，以支持数据的收集、转换、存储和查询等任务。

Apache Airflow是一个开源的任务调度和工作流管理平台。它提供了一种可编程的方式来定义、调度和监控数据流程和任务。使用Airflow，您可以创建有向无环图（DAG），定义任务之间的依赖关系和执行顺序，并按计划自动执行这些任务。Airflow支持多种任务类型，包括ETL（Extract-Transform-Load）、数据处理、模型训练等。它提供了一个用户友好的界面来监控任务的执行状态、调度历史和任务日志，同时还支持任务的重试、报警和调度策略等功能。

DorisDB（原Palo）：

DorisDB是一个开源的分布式列式存储数据库。它专注于高性能、低延迟的数据分析和查询。DorisDB以列式存储的方式组织数据，能够快速地处理大规模数据集和复杂查询。它支持高度可伸缩的架构，可以通过水平扩展来应对大规模数据和并发查询的需求。DorisDB还提供了分布式的数据存储和计算能力，可以进行数据的分片和副本管理，以保证数据的可靠性和高可用性。它广泛应用于数据仓库、大数据分析和实时查询等场景。

Sqoop、Kettle和FineReport是三个独立的数据工具，它们可以在数据处理和报表生成方面发挥各自的作用。

Sqoop：

Sqoop是一个开源的数据传输工具，用于在Hadoop和关系型数据库之间进行数据的导入和导出。Sqoop支持各种关系型数据库，如MySQL、Oracle、SQL Server等，可以将数据库中的数据导入到Hadoop生态系统中的分布式存储系统（如HDFS）中，也可以将Hadoop中的数据导出到关系型数据库中。Sqoop简化了大规模数据的导入导出过程，提供了命令行工具和API接口，方便用户进行数据迁移和集成。

Kettle（现称为Pentaho Data Integration）： AWS的一个类似kafka的中间价

Kettle是一款开源的ETL（Extract-Transform-Load）工具，用于数据的抽取、转换和加载。Kettle提供了可视化的界面和丰富的数据转换组件，用户可以通过拖拽和配置的方式定义数据处理的流程。Kettle支持多种数据源和目标，包括关系型数据库、文件、Web服务等，它可以对数据进行清洗、转换、合并等操作，并将处理后的数据加载到目标系统中。Kettle具有强大的数据集成和转换能力，可用于构建复杂的数据管道和ETL流程。

FineReport：

FineReport是一款企业级的报表设计和生成工具，它提供了可视化的报表设计界面和丰富的报表模板、图表和数据分析功能。FineReport支持多种数据源，包括关系型数据库、Excel、CSV等，用户可以通过简单的操作将数据源与报表进行关联，设计出具有数据分析和展示效果的报表。FineReport提供了灵活的报表布局和样式设置，支持多维数据分析、数据透视、参数查询等功能，使用户能够轻松生成各种复杂的报表和数据分析结果。

综合使用Sqoop、Kettle和FineReport，您可以实现从关系型数据库到Hadoop的数据导入、数据清洗和转换，然后使用FineReport生成可视化的报表和数据分析结果。Sqoop用于数据传输，将关系型数据库的数据导入到Hadoop中；Kettle用于数据处理和转换，对导入的数据进行清洗和转换操作；FineReport用于报表设计和生成，将处理后的数据展示为具有分析和可视化效果的报表。这样的组合可以满足大数据处理和报表生成的需求。

Scala和Java区别：

语法风格：Scala的语法更加紧凑和简洁，支持函数式编程的特性，而Java的语法相对更为繁琐和冗长。

静态类型 vs 动态类型：Java是一种静态类型语言，需要在编译时明确定义变量的类型，而Scala是一种混合了静态和动态类型特性的语言，它具有类型推断的能力，可以根据上下文自动推断变量的类型。

面向对象编程 vs 函数式编程：Java是一种面向对象的语言，重点在于类和对象的封装和继承关系，而Scala既支持面向对象编程，也支持函数式编程，函数是一等公民，可以作为参数传递和返回值使用。

并发编程：Scala内置了丰富的并发编程库和特性，例如Actor模型和不可变数据结构，使得编写并发程序更加简单和安全。相比之下，Java在并发编程方面的支持相对较弱，需要依赖于线程和锁等底层机制。

生态系统和库：Java拥有庞大的生态系统和丰富的第三方库支持，广泛应用于企业开发和大型系统。Scala的生态系统相对较小，但也有许多优秀的库和框架，特别适合于函数式编程和数据处理领域。

Go（又称为Golang）和Java区别：

性能和并发性：Go被设计为一种高性能的语言，具有轻量级的协程（goroutines）和内置的并发支持，使得编写高效并发程序变得简单。相比之下，Java在并发性方面的支持相对较弱，需要依赖于线程和锁等机制。

语言特性：Go采用了简洁而现代的语法，具有更轻量级的语言结构和更简洁的标准库。它强调可读性和简洁性，并提供了内置的错误处理机制。Java的语法相对较繁琐，但也提供了丰富的库和工具，适用于大规模的企业应用开发。

内存管理：Go具有自动内存管理（垃圾回收）功能，程序员无需手动管理内存分配和释放。而Java也提供自动内存管理，通过垃圾回收器（Garbage Collector）来管理内存。

静态类型 vs 动态类型：Java是一种静态类型语言，需要在编译时明确定义变量的类型，而Go是一种静态类型语言，但具有类型推断的能力，可以根据上下文自动推断变量的类型。

生态系统和库：Java拥有庞大的生态系统和广泛的第三方库支持，适用于各种应用场景，尤其在企业级开发中应用广泛。Go的生态系统相对较小，但也有一些优秀的库和工具，特别适合于构建网络服务和系统级应用。

TiDB是一个分布式的关系型数据库，具有强一致性、高并发、高可用、大规模数据性和自动扩展等特点，适用于处理大规模数据和高并发的场景。

分布式架构：TiDB采用分布式架构，可以水平扩展，实现高性能和高可用性。它使用Raft一致性算法来实现数据的分布式复制和故障恢复。

强一致性：TiDB保证数据的强一致性，这意味着当数据写入成功后，所有节点都可以立即读取到最新的数据。

支持SQL：TiDB兼容MySQL协议，可以使用标准的SQL语言进行数据操作和查询。它支持常见的SQL功能，如事务、索引、连接和聚合等。

自动水平扩展：TiDB可以根据负载情况自动扩展节点数量，以满足不断增长的数据和请求量。这使得TiDB非常适合处理大规模数据和高并发的场景。

分布式事务：TiDB支持分布式事务，可以保证在多个节点上的数据操作的原子性和一致性。

高可用性：TiDB使用Raft协议实现数据的分布式复制和故障恢复，可以在节点故障时自动进行主备切换，保证系统的高可用性。

数据分片和负载均衡：TiDB将数据分为多个分片，并通过自动负载均衡来均匀分布数据和请求，确保系统的高性能和可扩展性。

实时分析：TiDB支持实时分析查询，可以处理复杂的分析任务，并提供实时的查询结果。
