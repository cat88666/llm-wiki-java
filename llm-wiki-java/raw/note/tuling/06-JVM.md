### **一、JVM结构**

```
JVM 进程
 ├─main 线程启动
 ├─加载类（ClassLoader）
 │   ├─Bootstrap
 │   ├─Ext
 │   └─App
 ├─类生命周期
 │   加载 → 链接 → 初始化 → 使用 → 卸载
 ├─对象生命周期
 │   创建 → 不可见 → 不可达 → 可收集 → 终结 → 释放
 ├─引用类型
 │   强 / 软 / 弱 / 虚
 └─双亲委派
     父加载 → 父失败 → 子加载
```

### **二、JVM生命周期**

**1. JVM 实例生命周期**

- **诞生**：执行 `java` 命令 → 创建一个 JVM 实例
- **运行**：main() 线程启动，非守护线程运行时 JVM 存活
- **消亡**：所有非守护线程结束 或 执行 `System.exit()`

**2. 对象生命周期阶段**

1. **创建阶段**：分配内存 → 执行构造器（包括父类构造器） → 初始化实例变量
2. **可见 / 不可见阶段**：离开作用域后本地变量失效
3. **不可达阶段**：GC Root 不可达
4. **可收集阶段**：等待 GC 回收
5. **终结阶段**：执行 `finalize()`（仅一次）
6. **释放阶段**：内存空间被 GC 回收

**3. Java 四种引用类型**

| 引用类型 | 回收时机 | 典型用途 |
| --- | --- | --- |
| **强引用** | 永不回收（OOM 也不回收） | 普通对象 |
| **软引用（SoftRef）** | 内存不足才回收 | 缓存 |
| **弱引用（WeakRef）** | 一次 GC 必回收 | ThreadLocal、WeakHashMap |
| **虚引用（PhantomRef）** | 获取回收通知 | 管理堆外内存、Cleaner |

### **三、Java类生命周期**

5 大阶段：**加载 → 链接（验证/准备/解析） → 初始化 → 使用 → 卸载**

**1. 加载（Load）**

- 读取 `.class` 文件
- 将类元信息存入 **方法区**
- 在 **堆** 中创建一个 `java.lang.Class` 对象作为入口
- 类加载器完成此过程（Bootstrap/Ext/App/自定义）

**2. 链接（Link）**

**① 验证（Verify）：**保证字节码格式正确：文件格式、语法、符号引用合法性等

**② 准备（Prepare）：**为**静态变量**分配内存、赋默认值（0 / null）、**final static 常量 → 在准备阶段就赋真实值**

**③ 解析（Resolve）：**将符号引用替换为直接引用（指针/句柄）

**3. 初始化（Initialize）：**满足主动引用 → 执行：静态变量赋真实值、`static { }` 静态代码块、**先初始化父类，再初始化子类**

触发初始化的4种主动引用：

1. new 创建实例
2. 读取/修改静态变量
3. 调用静态方法
4. 反射（Class.forName）
5. main 类在启动时

不会触发初始化(被动引用)：访问父类静态字段（不会触发子类初始化）、定义类数组、访问编译期常量（final static）

**4. 使用（Use）：**正常使用类（静态方法、反射、实例化等）

**5. 卸载（Unload）：**需满足三条件：类的所有实例已被回收、定义该类的类加载器被回收、Class 对象不可达

### **四、类加载时机**

触发初始化的 7 种情况：

1. `new`
2. 访问静态变量
3. 调用静态方法
4. 反射 `Class.forName`
5. 初始化子类先初始化父类
6. JVM 启动加载 main 类
7. 非编译期常量的 final 静态字段

### **五、类加载器（ClassLoader）**

JVM 默认 3 个类加载器：**Bootstrap、Ext、App**

自定义类加载器：继承 `ClassLoader、`重写 `findClass()`

| 加载器 | 负责加载 |  |
| --- | --- | --- |
| **Bootstrap** | JDK 核心类库（rt.jar） | BootstrapClassLoader()引导类加载器 |
| **Ext** | JRE/ext 扩展包 | ExtClassLoader(扩展类加载器) |
| **App** | classpath 下的类 | AppClassLoader(应用类加载器) |

### **六、类加载机制**

**1. 全盘负责:** 一个类由哪个加载器加载，则该类引用的类也使用同一个加载器（除非显式指定）。

**2. 双亲委派:** 加载流程：自己不加载 → 交给父加载器,父加载器无法加载 → 子加载器再加载

- 保证核心 API 不被篡改（安全）、防止用户代码替换 JDK 类（如自定义 java.lang.String）
- 避免重复加载提高效率、确保类加载唯一性

**3. 缓存机制：**类加载后会缓存，提高性能。（类更新必须重启 JVM）
