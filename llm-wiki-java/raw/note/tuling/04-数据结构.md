**一、List**

| 数据结构 | 核心特性 | 时间复杂度 | 适用场景 | 备注 |
| --- | --- | --- | --- | --- |
| **ArrayList** | 动态数组、连续内存、CPU Cache 友好 | 访问 O(1)、插/删中间 O(n) | 内存碎片少，JVM 对对象数组优化较多 | 默认容量10,扩容 1.5 倍 |
| **LinkedList** | 双向链表、节点分散、通过指针连接 | 访问 O(n)、头尾 O(1) | 频繁头尾操作，节点额外指针开销GC压力大 | GC 压力大，不常用 |
| **CopyOnWriteArrayList** | 写时复制、读无锁 | 写 O(n)、读 O(1) | 读多写少 | 内存开销大 |

**二、Set**

| 数据结构 | 底层结构 | 时间复杂度 | 特点 | 适用场景 |
| --- | --- | --- | --- | --- |
| **HashSet** | HashMap Key | O(1) | 无序 | 最常用 Set |
| **TreeSet** | 红黑树 | O(log n) | 有序 | 排序、范围查询 |
| **LinkedHashSet** | Hash + 双链表 | O(1) | 保持插入顺序 | 需要顺序时 |

**三、Map**

**HashMap put：**`hashCode()`定位桶index -》桶为空 → 直接放; 桶非空→ 链表 / 红黑树 → 插入 / 更新；

**JDK 1.7 vs 1.8**

- 1.7：数组+链表(头插法，扩容转移元素时用头插法可能与并发rehash导致死循环) 扩容:新建2倍容量数组,遍历老数组上每个链表，重新rehash索引头插到新数组中;
- 1.8：数组+链表/红黑树(尾插法+冲突少链表/冲突严重树化) 扩容：数组长度翻倍 bit运算优化rehash过程，减少CPU开销，保持红黑树结构的有序性
- **树化条件:** 冲突少使用链表，当链表长度超过阈值(8)且数组≥64时转红黑树，当树节点数<6 退化为链表,避免小数据结构过于复杂。

| 数据结构 | 底层结构 | 时间复杂度 | 特点 | 适用场景 |
| --- | --- | --- | --- | --- |
| **HashMap** | 数组 + 链表 + 红黑树 | O(1)；树化后 O(log n) | 冲突 ≥ 8 且数组 ≥ 64 → 树化；线程不安全； | 主要Map 实现、hash 高位判断扩容迁移 |
| **LinkedHashMap** | HashMap + 双链表 | 查找 O(1) | 支持 LRU（accessOrder） | LRU 缓存 |
| **TreeMap** | 红黑树 | O(log n) | 有序 Key | 排序、范围扫描 |
| **ConcurrentHashMap** | 1.7：Segment；1.8：CAS + synchronized + 分段迁移 | O(1) | put 遇扩容会“帮助扩容” | 并发 Map 首选 |

**四、Queues**

| 队列类型 | 底层结构 | 特点 | 适用场景 |
| --- | --- | --- | --- |
| **ArrayBlockingQueue** | 数组 + 锁 | 固定容量，简单稳定 | 中等吞吐生产者消费者 |
| **LinkedBlockingQueue** | 链表 | 两把锁、吞吐更高 | 高频生产/消费场景 |
| **SynchronousQueue** | 无容量 | 生产必须等消费，直接移交 | 线程池（直接提交） |
| **DelayQueue / PriorityQueue** | 堆结构 | 延迟任务 / 优先级任务 | 定时任务、调度 |

**五、核心原理**

| 原理 | 核心机制 | 关键特性 | 优缺点 | 使用场景 |
| --- | --- | --- | --- | --- |
| **哈希表(Hash)** | 数组 + 链表/树 + 扰动函数 | O(1)查找、树化(>=8)、扩容逻辑(高位迁移) | 极快；但扩容耗时，可触发链表退化 | Map/Set 基础结构，缓存、索引 |
| **红黑树(RB-Tree)** | 弱平衡二叉树，少旋转 | O(log n)、稳定、工业级性能 | 插入删除快于AVL，结构稳定 | TreeMap / TreeSet / HashMap冲突树化 |
| **跳表(SkipList)** | 链表 + 多层索引，随机层高 | 类二分查找、O(log n)、范围查询强 | 实现简单，跳层概率算法优秀，并发场景下做分段 / 无锁优化 | Redis ZSet、ConcurrentSkipListMap |
| **Copy-On-Write(COW)** | 写时复制，读用旧数组 | 读无锁，读快、写慢 | 内存占用大，写延迟高 | 配置、黑名单、白名单（读多写少） |
| **CAS(Compare And Swap)** | compare + swap | 自旋、锁无竞争快、ABA问题 | 高并发性能极强，ABA 要版本号 | AtomicXXX、CHM、AQS、无锁原子操作 |
| **Fail-Fast** | 结构修改检出 | 快速失败（抛异常） | 保证一致性，避免“脏迭代” | HashMap、ArrayList 迭代中修改 |
| **弱一致性** | 修改可同时读 | 不保证实时可见，不 fail-fast | 高并发性能优先，不保证一致快照 | ConcurrentHashMap、并发集 |

**六、树**

| 特性 | **BST（普通二叉搜索树）** | **AVL（平衡二叉树）** | **红黑树（弱平衡树）** |
| --- | --- | --- | --- |
| **结构类型** | 单纯按大小插入的 BST | 强平衡 BST | 弱平衡 BST（着色+旋转） |
| **平衡要求** | 无平衡要求 | **高度差 ≤ 1**（强平衡） | “最长路径 ≤ 最短路径 * 2”（弱平衡） |
| **最坏高度** | **O(n)**（退化为链表） | **O(log n)**（严格） | **O(log n)**（较宽松） |
| **查找性能** | 波动大，最差 O(n) | 最稳，最接近完美平衡 | 稍弱于AVL，但更稳定 |
| **插入/删除成本** | O(1) 链表式增长 | **旋转频繁、成本高** | **旋转少，插入最优** |
| **旋转次数** | 无 | **多**（严格平衡需要频繁旋转） | **少**（弱平衡减少旋转） |
| **维护难度** | 简单 | 复杂 | 较复杂但工程成熟 |
| **内存占用** | 最少 | 较高（存高度） | 较高（存颜色） |
| **适用场景** | 小数据量，简单需求 | **查多写少、高速读需求** | **通用场景，写入多，高并发** |
| **工业级应用** | 很少直接用 | 数据库部分索引结构 | **TreeMap / TreeSet / HashMap 冲突树化 / Linux 内核** |
| **总结** | *可能退化，不可控* | *追求最稳查找性能* | *少旋转 + 强性能 = 工业标配* |
