## 1. 时间复杂度速查

| 复杂度 | 名称 | 典型算法 | n=10^6 耗时估算 |
| --- | --- | --- | --- |
| O(1) | 常数 | 哈希表查找 | 极快 |
| O(log n) | 对数 | 二分查找、平衡树 | 约20次操作 |
| O(n) | 线性 | 线性扫描 | 10^6 次操作 |
| O(n log n) | 线性对数 | 归并排序、快速排序 | 约2×10^7 次操作 |
| O(n²) | 平方 | 冒泡排序、暴力匹配 | 10^12 次，不可接受 |

## 2. 核心数据结构

### 2.1 数组与链表

**数组**：随机访问 O(1)，插入/删除 O(n)，内存连续，缓存友好。

**链表**：随机访问 O(n)，插入/删除 O(1)（已知节点），内存不连续。

**高频题型**：反转链表、合并有序链表、链表是否有环（快慢指针）、找链表中点、LRU 缓存（HashMap + 双向链表）。

**Floyd 判圈算法（快慢指针）**：

- 快指针每次走2步，慢指针每次走1步
- 有环则快慢指针必然相遇
- 相遇后，将一个指针移到头节点，两指针同速前进，再次相遇点即为环的入口

### 2.2 栈与队列

**栈**：LIFO，用于括号匹配、表达式求值、DFS、函数调用栈。

**队列**：FIFO，用于 BFS、任务调度。

**单调栈**：栈内元素保持单调递增或递减，用于求下一个更大/更小元素、柱状图最大矩形。

**高频题型**：用两个栈实现队列、最小栈（辅助栈存最小值）、接雨水（单调栈/双指针）。

### 2.3 树

**二叉树遍历**：

- 前序（根-左-右）：序列化树结构
- 中序（左-根-右）：BST 中序遍历得到有序序列
- 后序（左-右-根）：计算目录大小、删除树节点
- 层序（BFS）：按层处理，求树的高度、锯齿形遍历

**二叉搜索树（BST）**：左子树所有节点 < 根 < 右子树所有节点，中序遍历有序，查找/插入/删除平均 O(log n)，最坏 O(n)（退化为链表）。

**平衡二叉树（AVL）**：任意节点左右子树高度差 ≤ 1，保证 O(log n) 操作，但旋转操作频繁。

**红黑树**：近似平衡，插入/删除最多3次旋转，查找/插入/删除均为 O(log n)。Java 的 TreeMap、HashMap（链表转树）使用红黑树。

**高频题型**：二叉树最大深度、路径和、对称二叉树、最近公共祖先、序列化与反序列化。

### 2.4 堆（优先队列）

**堆的性质**：完全二叉树，父节点 ≤ 子节点（小顶堆）或 ≥ 子节点（大顶堆）。

**堆的操作**：插入 O(log n)，删除堆顶 O(log n)，建堆 O(n)。

**高频应用**：

- Top K 问题：维护大小为 K 的小顶堆，遍历数组，比堆顶大则替换
- 合并 K 个有序链表：K 个链表头节点入堆，每次取最小节点
- 数据流中位数：大顶堆存较小的一半，小顶堆存较大的一半，保持两堆大小差 ≤ 1

### 2.5 哈希表

**哈希冲突解决**：

- 链地址法（Chaining）：冲突的元素存入链表（Java HashMap）
- 开放地址法（Open Addressing）：冲突时寻找下一个空槽（线性探测/二次探测）

**高频题型**：两数之和（HashMap 存补数）、字母异位词分组（排序后的字符串作 key）、最长无重复子串（滑动窗口 + HashMap）。

## 3. 核心算法

### 3.1 排序算法

| 算法 | 平均时间 | 最坏时间 | 空间 | 稳定 |
| --- | --- | --- | --- | --- |
| 冒泡排序 | O(n²) | O(n²) | O(1) | 是 |
| 选择排序 | O(n²) | O(n²) | O(1) | 否 |
| 插入排序 | O(n²) | O(n²) | O(1) | 是 |
| 归并排序 | O(n log n) | O(n log n) | O(n) | 是 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | 否 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | 否 |

**快速排序优化**：随机选择 pivot（避免最坏情况）、三路划分（处理大量重复元素）。

**Java Arrays.sort() 实现**：基本类型用双轴快速排序（DualPivotQuicksort），对象类型用 TimSort（归并排序变体，稳定）。

### 3.2 二分查找

**标准模板**：

```java
int left = 0, right = nums.length - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;  // 防止溢出
    if (nums[mid] == target) return mid;
    else if (nums[mid] < target) left = mid + 1;
    else right = mid - 1;
}
return -1;
```

**变体**：查找第一个 ≥ target 的位置（lower_bound）、最后一个 ≤ target 的位置（upper_bound）。

**应用场景**：有序数组查找、旋转有序数组查找、求平方根、搜索插入位置。

### 3.3 双指针与滑动窗口

**对撞指针**：两端向中间移动，用于有序数组的两数之和、三数之和、接雨水。

**快慢指针**：链表判环、找中点、删除倒数第 N 个节点。

**滑动窗口**：维护一个窗口，通过移动左右边界解决子串/子数组问题。

- 最长无重复子串：右指针扩展，遇到重复字符时左指针收缩
- 最小覆盖子串：右指针扩展直到覆盖所有字符，左指针收缩找最小窗口

### 3.4 动态规划

**解题步骤**：

1. 定义状态：`dp[i]` 表示什么
2. 状态转移方程：`dp[i]` 如何由之前的状态推导
3. 初始化：边界条件
4. 计算顺序：确保计算 `dp[i]` 时所需的状态已计算

**经典题型**：

| 题目 | 状态定义 | 转移方程 |
| --- | --- | --- |
| 最长递增子序列（LIS） | dp[i]=以nums[i]结尾的LIS长度 | dp[i]=max(dp[j]+1) for j&lt;i且nums[j]&lt;nums[i] |
| 最长公共子序列（LCS） | dp[i][j]=s1前i字符与s2前j字符的LCS | s1[i]==s2[j]: dp[i][j]=dp[i-1][j-1]+1; 否则max(dp[i-1][j],dp[i][j-1]) |
| 0-1背包 | dp[i][j]=前i件物品容量j的最大价值 | dp[i][j]=max(dp[i-1][j], dp[i-1][j-w[i]]+v[i]) |
| 最大子数组和 | dp[i]=以nums[i]结尾的最大子数组和 | dp[i]=max(nums[i], dp[i-1]+nums[i]) |

### 3.5 图算法

**BFS（广度优先搜索）**：层序遍历，求最短路径（无权图），用队列实现。

**DFS（深度优先搜索）**：递归或栈实现，用于连通性判断、拓扑排序、全排列。

**并查集（Union-Find）**：高效处理连通性问题，路径压缩 + 按秩合并后近似 O(1)。

```java
int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]);  // 路径压缩
    return parent[x];
}
void union(int x, int y) {
    int px = find(x), py = find(y);
    if (px != py) parent[px] = py;
}
```

**拓扑排序**：有向无环图（DAG）的线性排序，用于任务调度、课程依赖。Kahn 算法（BFS + 入度）。

## 4. 大厂高频面试题

### 4.1 LRU 缓存

**要求**：O(1) 时间复杂度的 get 和 put 操作。

**实现**：HashMap（O(1) 查找）+ 双向链表（O(1) 移动节点）。

- get：找到节点，移到链表头部
- put：存在则更新并移到头部；不存在则新建节点放头部，若超容量则删除尾部节点

Java 可用 `LinkedHashMap`（`accessOrder=true`）直接实现。

### 4.2 Top K 问题

**方法一（小顶堆）**：维护大小为 K 的小顶堆，遍历数组，比堆顶大则替换堆顶并重新堆化。时间 O(n log k)，空间 O(k)。

**方法二（快速选择）**：基于快速排序的 partition，平均 O(n)，最坏 O(n²)。

### 4.3 海量数据处理

**100亿个整数，找出出现次数最多的前100个**：

1. 哈希分片：将数据按 `hash(num) % 1000` 分成1000个文件
2. 每个文件用 HashMap 统计频次
3. 用小顶堆合并各文件的 Top 100

**两个大文件（各100亿行URL），找出共同的URL**：

1. 哈希分片：同一 URL 必然分到相同编号的文件
2. 对每对同编号文件求交集（用 HashSet）

**布隆过滤器**：判断元素是否在集合中，有一定误判率（假阳性），但无假阴性，空间效率极高。适合大规模黑名单过滤、缓存穿透防护。
