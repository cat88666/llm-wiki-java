# 10-ELK概念

所有者: junk01

**ES全文检索**

通过一个程序扫描文本中的每一个单词，针对单词建立索引，并保存该单词在文本中的位置、以及出现的次数用户查询时，通过之前建立好的索引来查询，将索引中单词对应的文本位置、出现的次数返回给用户，因为有了具体文本的位置，所以就可以将具体内容读取出来了

**ES倒排索引**

索引就类似于目录，平时我们使用的都是索引，都是通过主键定位到某条数据，那么倒排索引呢，刚好相反，数据对应到主键。ES底层在检索时底层使用的就是倒排索引

ES要掌握什么：

1. 使用：查询语法，聚合操作语法，理解相关性算分（文档匹配度）

2. 优化： 文档建模，语法选择，ES集群架构优化，数据处理

**经典的ELK**

经典的ELK主要是由Filebeat + Logstash + Elasticsearch + Kibana（早期的ELK只有Logstash + Elasticsearch + Kibana）

**Logstash数据传输原理**

1. 数据采集与输入：

Logstash支持各种输入选择，能够以连续的流式传输方式，轻松地从日志、指标、Web应用以及数据存储中采集数据。

2. 实时解析和数据转换：

通过Logstash过滤器解析各个事件，识别已命名的字段来构建结构，并将它们转换成通用格式，最终将数据从源端传输到存储库中。

3. 存储与数据导出：

Logstash提供多种输出选择，可以将数据发送到指定的地方。Logstash通过管道完成数据的采集与处理，管道配置中包含input、output和filter（可

选）插件，input和output用来配置输入和输出数据源、filter用来对数据进行过滤或预处理

什么是Elasticsearch（ES）？它的主要特点是什么？

答案：Elasticsearch是一个开源的分布式搜索和分析引擎，用于存储、搜索和分析大规模的数据。其主要特点包括高度可扩展性、实时性、全文搜索、分布式架构、多租户支持和强大的数据分析功能。

请解释一下Elasticsearch的倒排索引是什么？

答案：倒排索引是一种用于快速查找文档的数据结构。在传统的索引中，我们根据关键词去查找文档，而在倒排索引中，我们根据文档去查找关键词。它通过记录每个关键词出现在哪些文档中，以及在文档中的位置，实现快速的全文搜索。

如何在Elasticsearch中进行数据的索引和检索操作？

答案：在Elasticsearch中，我们可以使用RESTful API来进行数据的索引和检索操作。使用HTTP协议发送请求，并根据API提供的参数来指定索引、类型、查询条件等信息。例如，使用PUT /index/_doc/1可以将文档索引到名为index的索引中，使用GET /index/_search可以执行查询操作。

请解释一下Elasticsearch的分布式架构是如何工作的？

答案：Elasticsearch采用分布式架构，数据被分割成多个分片，每个分片被存储在不同的节点上。每个节点都知道索引中的所有分片的位置。当进行数据的索引或搜索操作时，请求会被路由到相应的分片上进行处理，并将结果合并返回给客户端。

如何保证Elasticsearch的高可用性？

使用主备复制：每个分片都有一个主分片和多个副本分片，主分片处理写操作，副本分片用于备份和读操作。

自动故障转移：当主节点不可用时，副本分片会自动选举出新的主节点，确保系统的正常运行。

数据复制：将数据复制到不同的节点上，以防止节点故障导致数据丢失。

什么是Elasticsearch索引的刷新和合并？

答案：索引的刷新是将内存中的数据写入磁盘，使得数据对搜索可见。默认情况下，Elasticsearch每秒自动刷新索引，也可以手动触发刷新操作。索引的合并是将多个段（segments）合并成一个较大的段，减少磁盘上的碎片并提高性能。

如何提高Elasticsearch的查询性能？

设计合理的索引结构，包括合适的字段类型和分词器。

使用查询缓存和过滤器缓存来缓存频繁使用的查询和过滤器结果。

优化查询语句，避免不必要的复杂性和性能开销。

使用适当的分片数量和副本数量，以平衡负载和提高并发性能。

什么是Elasticsearch的聚合（Aggregations）功能？如何使用它？

Elasticsearch的聚合功能用于在搜索结果上进行统计和分析操作，如计数、求和、平均值、最大值、最小值等。我们可以使用聚合查询来定义聚合操作，并根据需要添加多个聚合子句。聚合结果以层级结构返回，可以进行嵌套和多级聚合。

如何进行Elasticsearch的数据备份和恢复？

可以使用Elasticsearch的快照和恢复功能来进行数据备份和恢复。通过创建快照，可以将索引和分片的数据保存到外部存储库中。在需要恢复数据时，可以使用快照进行恢复操作，将数据从存储库还原到Elasticsearch集群中。

请解释一下Elasticsearch的分词器（Tokenizer）和分析器（Analyzer）的作用和区别。

答案：分词器用于将文本字符串分割成一个个单词（词条），并去除停用词和特殊字符。分析器则包括分词器和一系列的过滤器，用于对文本进行处理、规范化和转换。分词器执行最基本的词条分割操作，而分析器在此基础上添加了更多的处理步骤，如大小写转换、词干提取、同义词替换等。
