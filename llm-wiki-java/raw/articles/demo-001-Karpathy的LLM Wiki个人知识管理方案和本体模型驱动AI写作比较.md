## LLM Wiki —— Karpathy 的个人知识管理方案

### 核心思想：从"检索"到"编译"
传统的 RAG 方案（NotebookLM、ChatGPT 文件上传等）每次提问时都让 LLM 从原始文档中重新检索和拼凑答案，没有任何积累。

Karpathy 提出的模式截然不同：LLM 不仅仅是在查询时检索，而是增量地构建并维护一个持久化的 Wiki——一个结构化、相互链接的 Markdown 文件集合。每当你加入新资料，LLM 会读取它、提取关键信息，并将其整合进已有 wiki——更新实体页面、修订主题摘要、标注新数据与旧观点的矛盾之处。知识被"编译"一次，然后持续保持最新，而不是每次查询时重新推导。

这是一个复利效应：交叉引用已经就位，矛盾已经被标记，综合分析已经涵盖你读过的所有内容。随着每增加一个来源或每提出一个问题，wiki 就变得更加丰富。

### 三层架构

原始资料层（Raw Sources）：你精心收集的原始文档——文章、论文、图片、数据。这些是不可修改的，LLM 只读取它们，永不修改。Wiki 层：一个 LLM 生成的 Markdown 文件目录，包含摘要、实体页面、概念页面、对比分析、概述与综合。LLM 完全拥有这一层。Schema（配置文件）：一个告诉 LLM wiki 如何组织、遵循什么约定以及执行什么工作流的文档（如 CLAUDE.md 或 AGENTS.md）。这是核心配置文件，让 LLM 成为一个有纪律的 wiki 维护者而非普通聊天机器人。 github

### 三种核心操作
Ingest（摄入）：你放入一个新资料并让 LLM 处理。LLM 阅读来源，与你讨论关键要点，在 wiki 中写一个摘要页，更新索引，更新相关实体和概念页面，并追加一条日志记录。一个资料可能触及 10-15 个 wiki 页面。Query（查询）：对 wiki 提问。LLM 搜索相关页面，读取它们，并给出带引用的回答。关键洞见：好的回答可以作为新页面归档回 wiki，这样你的探索就像摄入的资料一样在知识库中不断积累。Lint（健康检查）：定期让 LLM 检查 wiki，寻找页面间的矛盾、被新资料推翻的陈旧说法、没有入站链接的孤立页面等。

### 实用工具建议

实际使用时，可以一边打开 LLM Agent，一边打开 Obsidian。LLM 根据对话做出编辑，你实时浏览结果——跟随链接、查看图谱视图、阅读更新后的页面。Obsidian 是 IDE；LLM 是程序员；Wiki 是代码库。

index.md 是内容导向的，是 wiki 中所有内容的目录；log.md 是时间线，记录所有摄入、查询、检查操作，是 append-only 的历史记录。

### 为什么这个方案能成立？

维护知识库的繁琐部分不是阅读或思考，而是记账工作：更新交叉引用、保持摘要最新、标注新数据与旧主张的矛盾、维护数十个页面的一致性。人类之所以放弃 wiki，是因为维护负担增长速度超过了价值。LLM 不会感到厌倦，不会忘记更新交叉引用，能在一次操作中触及 15 个文件。

这个想法在精神上与 Vannevar Bush 1945 年提出的 Memex 相关——一个具有文档间关联链接的个人知识存储。Bush 当时无法解决的问题是：谁来做维护？LLM 解决了这个问题。

### 应用场景

个人成长与健康追踪、学术研究深潜、读书笔记（类比托尔金粉丝 wiki）、团队内部知识库（由 Slack 线程、会议记录、项目文档喂养）、竞品分析、尽职调查、旅行规划、课程笔记，乃至任何需要随时间积累知识的场景。

简而言之，这个方案的精髓是：把 LLM 从一个"问答机器"变成一个"主动维护你知识库的协作者"，而你只需要负责提供原材料和提出好问题。你对这个方案有什么具体想法或想落地实现的方向吗？

## 完整落地方案

### 第一步：整理目录结构
你现有的文章作为 raw/，新建一个平行的 wiki/ 目录：

```text
my-knowledge/
├── raw/ ← 你现有的1000篇（只读）
│ ├── tech/
│ ├── books/
│ ├── journal/
│ └── ...
├── wiki/ ← AI 维护的结构化知识库（新建）
│ ├── index.md ← 所有页面目录
│ ├── log.md ← 操作记录
│ ├── concepts/ ← 主题/概念页
│ ├── entities/ ← 人物/项目/书籍页
│ └── synthesis/ ← 综合分析页
└── CLAUDE.md ← Schema：告诉AI如何工作
```

### 第二步：写好 CLAUDE.md（这是最关键的文件）
这是驱动整个系统的"宪法"，放在根目录，告诉 Claude Code 如何操作：

```markdown
# My Knowledge Wiki — Schema

## 目录结构约定
- raw/ 是只读原始来源，永不修改
- wiki/ 是你维护的知识库，所有文件均为 Markdown
- wiki/index.md 是所有页面的目录，每次操作后更新
- wiki/log.md 记录所有操作，append-only，格式：## [日期] 操作类型 | 标题

## 摄入流程（Ingest）
1. 读取 raw/ 下指定文件
2. 提取关键概念、实体、观点
3. 在 wiki/concepts/ 和 wiki/entities/ 创建或更新相关页面
4. 写 wiki/summaries/[文件名].md 摘要
5. 更新 wiki/index.md 和 wiki/log.md

## 页面格式
每个 wiki 页面包含：
- 一句话定义
```
