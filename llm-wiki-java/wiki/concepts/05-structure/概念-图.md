# 图

> 图由顶点（Vertex）和边（Edge）组成，能表达任意"多对多"关系——树是图的特殊情况（无环连通图）；图的两种核心遍历 DFS 和 BFS 是大量算法的骨架。

## 一、第一性原理

链表表达一对一，树表达一对多（父→子），而现实世界中大量关系是多对多（社交网络、交通路网、依赖关系）。图的存在理由：**建模任意拓扑关系的通用结构，顶点 + 边是最小必要表示**。

## 二、核心机制

### 图的分类

| 维度 | 类型 | 说明 |
|------|------|------|
| 边方向 | 有向图 | A→B 不代表 B→A（如网页跳转、任务依赖）|
| | 无向图 | A-B 双向可达（如社交好友、道路）|
| 边权重 | 有权图 | 边附带权重（距离、成本）|
| | 无权图 | 边仅表示连接关系 |
| 是否含环 | 有环图 | 可以从一个节点出发回到自身 |
| | DAG（有向无环图）| 拓扑排序的基础（任务调度、依赖解析）|

**树是图的特例**：无环 + 连通 + N 个节点 N-1 条边。

### 存储方式

**邻接矩阵**（适合稠密图）：
- 二维数组 `matrix[i][j]` = 边权重（或 0/1）
- 查询两点是否相邻：O(1)
- 空间：O(V²)，稀疏图浪费严重

**邻接表**（适合稀疏图）：
- 每个顶点维护一个邻居链表
- 空间：O(V + E)，存实际边数
- 遍历邻居：O(degree)

### 两种遍历

**DFS（深度优先）**：
```
用栈（或递归调用栈）
→ 一路走到底再回溯
→ 找连通分量、拓扑排序、检测环、迷宫求解
```

**BFS（广度优先）**：
```
用队列
→ 一层层向外扩展
→ 无权图最短路径、社交网络六度分隔、最少步数
```

### 经典算法依赖图

| 算法 | 图类型 | 数据结构 |
|------|--------|---------|
| Dijkstra 最短路径 | 有权有向图 | 小顶堆 + 邻接表 |
| 拓扑排序 | DAG | BFS（入度为 0 的节点入队）|
| 并查集 | 无向图连通性 | 数组（路径压缩）|
| Prim/Kruskal 最小生成树 | 无向加权图 | 堆/排序 |

## 三、核心使用原则

1. **邻接矩阵 vs 邻接表**：矩阵查边快 O(1)，但稀疏图空间浪费；表空间省，但查边 O(degree)
2. **DFS vs BFS**：DFS 空间 O(深度)，BFS 空间 O(宽度）；无权最短路必须用 BFS
3. **有向图的不对称性**：入度/出度需分别维护；强连通分量比树复杂得多

## 四、与其他概念的关系

- 包含 [[概念-Java集合]]：BFS 依赖队列，DFS 依赖栈，都是线性结构
- 依赖 [[机制-优先队列]]：Dijkstra 用小顶堆维护待处理节点的最短路径
- 树是图的特殊形式：所有树结构（红黑树、B+树、Trie）都是无环连通有向图

## 五、应用边界

**社交图谱**：好友推荐（BFS 找 N 度好友）；影响力分析（PageRank）

**路径规划**：GPS 导航（Dijkstra）；最少中转次数（BFS）

**工程依赖**：Maven 依赖解析（DAG 拓扑排序，检测循环依赖）；微服务调用链

**不适合**：纯层次结构（用树）；一对一序列（用链表/数组）；图计算规模超大时需要专用图数据库（Neo4j）或图计算框架（GraphX）
