# 网络安全

> 网络安全围绕身份可信、输入可信、传输可信、数据可信和服务可用构建防线，Java 后端重点关注认证授权、注入防护、Web 攻击、密码学使用和生产防护边界。

## 一、第一性原理

安全问题本质是信任边界失效：

| 边界 | 攻击方式 | 防御方向 |
| --- | --- | --- |
| 输入不可信 | SQL 注入、XSS、命令注入 | 参数化、校验、编码、沙箱 |
| 身份不可信 | 伪造登录态、撞库、Token 窃取 | 认证、MFA、会话管理、风控 |
| 权限不可信 | 水平越权、垂直越权 | RBAC/ABAC、资源归属校验 |
| 传输不可信 | 中间人、DNS 劫持 | HTTPS、证书校验、HSTS |
| 服务可用性不可信 | DDoS、刷接口、爆破 | 限流、熔断、WAF、清洗 |

目标不是“绝对安全”，而是在机密性、完整性、可用性之间做成本可控的分层防御。

## 二、核心机制

### 密码学基础

| 操作 | 解决什么 | 是否可逆 | 典型算法 | Java 后端结论 |
| --- | --- | --- | --- | --- |
| 加密/解密 | 机密性 | 可逆 | AES/SM4、RSA/SM2 | 敏感数据传输/存储 |
| 加签/验签 | 完整性、来源可信 | 不用于还原原文 | RSA/ECDSA/SM2 + Hash | 支付回调、开放 API |
| 哈希摘要 | 完整性校验、不可逆摘要 | 不可逆 | SHA-256/SM3、bcrypt、Argon2 | 密码不能用 MD5 |

MD5 是摘要算法，不是加密算法；也不适合存密码。密码存储应使用 bcrypt、Argon2、PBKDF2 这类慢哈希并加盐。

### SQL 注入

| 错误做法 | 风险 | 正确做法 |
| --- | --- | --- |
| 拼接 SQL 字符串 | 用户输入改变 SQL 结构 | PreparedStatement、MyBatis `#{}` |
| MyBatis `${}` 接用户输入 | 直接文本替换 | 只用于白名单字段，如排序列 |
| 只依赖前端校验 | 可被绕过 | 服务端校验 + DB 最小权限 |

预编译能防注入的关键是：SQL 结构先编译，参数作为数据绑定，不再参与 SQL 语法解析。

### XSS 与 CSRF

| 攻击 | 本质 | 防御 |
| --- | --- | --- |
| XSS | 恶意脚本在用户浏览器执行 | 输出编码、CSP、HttpOnly Cookie、输入净化 |
| CSRF | 借用户已登录态发起伪造请求 | CSRF Token、SameSite Cookie、Referer/Origin 校验 |

XSS 利用的是“页面执行了不可信脚本”；CSRF 利用的是“浏览器自动携带 Cookie”。二者不是同一类问题。

### 越权访问

| 类型 | 例子 | 防御 |
| --- | --- | --- |
| 垂直越权 | 普通用户调用管理员接口 | 角色/权限校验，默认拒绝 |
| 水平越权 | 用户 A 查询用户 B 的订单 | 资源 owner 校验，不信任请求里的 userId |
| 枚举攻击 | 遍历递增 ID 批量拉数据 | ID 混淆、访问频控、审计告警 |

后端必须基于当前登录主体做权限判断，不能相信前端传来的用户 ID、角色或租户 ID。

### 中间人、DNS 与 DDoS

| 攻击 | 目标 | 防御 |
| --- | --- | --- |
| 中间人攻击 | 窃听/篡改传输数据 | HTTPS、证书校验、HSTS、证书固定 |
| DNS 劫持/污染 | 把域名解析到错误 IP | DoH/DoT、DNSSEC、可信 DNS、结果校验 |
| DDoS | 耗尽带宽、连接数、CPU | CDN、Anycast、清洗、WAF、限流、降级 |

DDoS 不能只靠应用层解决，大流量攻击需要网络层和云厂商清洗能力。

### 国密算法

| 算法 | 类型 | 对应国际算法 | 用途 |
| --- | --- | --- | --- |
| SM2 | 非对称/签名 | ECC/RSA | 签名、密钥交换 |
| SM3 | 哈希摘要 | SHA-256 | 摘要、签名前哈希 |
| SM4 | 对称加密 | AES | 数据加密 |
| SM9 | 标识密码 | 无直接等价 | 基于身份的密码体系 |

政务、金融、国企等场景可能有国密合规要求，不能只按通用互联网方案选型。

## 三、Java 核心使用

| 场景 | Java 常用方案 | 注意点 |
| --- | --- | --- |
| 认证授权 | Spring Security、Sa-Token、OAuth2、JWT | 区分认证、授权、会话管理 |
| SQL 防注入 | JDBC PreparedStatement、MyBatis `#{}`、JPA | 禁止拼接用户输入 |
| 密码存储 | BCryptPasswordEncoder、Argon2 | 不存明文、MD5、可逆密文 |
| 接口签名 | HMAC、RSA/SM2 签名 | 时间戳、nonce、防重放 |
| CORS | Spring MVC/Gateway CORS 配置 | 不要无脑 `*` + credentials |
| Cookie 安全 | HttpOnly、Secure、SameSite | 防 XSS 窃取和 CSRF |
| 敏感数据 | 加密、脱敏、审计 | 密钥管理比算法更关键 |

安全逻辑应尽量在网关、过滤器、拦截器、框架中统一收口，不要散落在每个业务方法里。

## 四、核心使用原则

1. **默认拒绝**：没有明确授权就拒绝访问。
2. **最小权限**：账号、数据库、服务、Token 都只给必要权限。
3. **输入校验 + 输出编码**：输入防注入，输出防 XSS。
4. **认证不等于授权**：登录成功只说明身份可信，不说明有权访问资源。
5. **敏感操作要防重放**：时间戳、nonce、签名、幂等号缺一不可。
6. **密钥不能进代码仓库**：使用 KMS、环境变量、密钥配置中心，并支持轮换。
7. **安全要可审计**：登录、授权失败、越权尝试、敏感数据访问都要留痕。

## 五、使用案例

| 场景 | 关键防护 | 说明 |
| --- | --- | --- |
| 用户登录 | bcrypt/Argon2、限频、MFA | 防拖库、撞库、爆破 |
| 查询订单 | owner 校验、租户校验 | 防水平越权 |
| 管理后台 | RBAC、操作审计、IP 白名单 | 防垂直越权和误操作 |
| 支付回调 | 验签、时间戳、nonce、幂等 | 防伪造、防重放、防重复入账 |
| 文件上传 | 类型校验、大小限制、隔离存储 | 防脚本上传和资源耗尽 |
| API 网关 | 认证、限流、黑名单、WAF | 安全能力前置统一治理 |

## 六、综合对比

| 对比 | 结论 |
| --- | --- |
| 加密 vs 签名 vs 哈希 | 加密保护机密性，签名证明来源和完整性，哈希生成不可逆摘要 |
| XSS vs CSRF | XSS 执行恶意脚本，CSRF 借用户 Cookie 发请求 |
| 水平越权 vs 垂直越权 | 水平是访问同级他人资源，垂直是低权限访问高权限功能 |
| Session vs Token | Session 服务端有状态、可主动失效；Token 易扩展但失效治理更复杂 |
| AES/SM4 vs RSA/SM2 | 对称加密快，非对称适合密钥交换和签名 |
| 限流 vs DDoS 防护 | 应用限流处理业务层压力，大流量 DDoS 需要网络层清洗 |

## 七、生产风险

| 风险 | 表现 | 处理原则 |
| --- | --- | --- |
| 密钥泄露 | 签名伪造、数据可解密 | KMS、轮换、权限隔离、禁止入库入仓 |
| Token 长期有效 | 账号被盗后长期可用 | 短 TTL、刷新令牌、黑名单、设备管理 |
| 越权漏洞 | 用户访问他人数据 | 每次按当前主体校验资源归属 |
| CORS 误配 | 任意站点携带凭证访问 | 精确 allow origin，谨慎 credentials |
| SQL 注入 | 数据泄露、删库 | 预编译、白名单、最小 DB 权限 |
| XSS | Cookie/Token 被盗、页面劫持 | 输出编码、CSP、HttpOnly |
| 撞库爆破 | 登录接口 QPS 异常 | 限频、验证码、MFA、风控 |
| 审计缺失 | 事故后无法追踪 | 敏感操作日志、不可抵赖记录 |

## 八、与其他概念的关系

- **[[概念-计算机网络]]**：HTTPS、DNS、Cookie、跨域是网络安全的基础边界。
- **[[概念-OAuth2]]**：OAuth2 解决第三方授权和 API 访问授权，不等同于完整安全体系。
- **[[机制-数据加密与脱敏]]**：网络安全偏攻防与访问边界，数据加密脱敏偏数据保护落地。
- **[[概念-幂等设计]]**：支付回调、防重放、重复请求处理都依赖幂等语义。

## 九、应用边界

- 安全不能靠单点能力解决，必须组合认证、授权、输入校验、加密、限流、审计。
- 内网服务也不能默认安全，内网横向移动、误配置和凭证泄露同样常见。
- WAF、网关、框架只能降低风险，不能替代业务侧资源归属校验。
- 合规场景要单独确认国密、等保、审计留存、数据出境和隐私保护要求。
