# 包装类与自动拆装箱

> 用对象"包裹"基本类型，使基本类型能参与面向对象体系（泛型集合、null 语义、反射）；自动拆装箱让两者在代码层透明转换。

## 第一性原理

Java 泛型和集合框架要求 `Object` 类型，`int`、`long` 等基本类型不继承 `Object`，无法直接放入 `List<Integer>` 或用作 `Map` key。包装类的存在理由：**让基本类型具备对象身份，参与需要对象的场景，同时保留基本类型的值语义**。

## 核心机制

### 8 种基本类型与包装类

| 基本类型 | 包装类 | 占用字节 | 默认值差异 |
|---------|--------|---------|-----------|
| `int` | `Integer` | 4 | 0 vs null |
| `long` | `Long` | 8 | 0L vs null |
| `double` | `Double` | 8 | 0.0 vs null |
| `boolean` | `Boolean` | 1 | false vs null |
| `char` | `Character` | 2 | '\u0000' vs null |
| `byte/short/float` | 对应包装类 | — | — |

### 自动装箱/拆箱原理

编译器语法糖，实际调用：
```java
Integer i = 42;          // → Integer.valueOf(42)
int n = i;               // → i.intValue()
```

### Integer 缓存（重要陷阱）

`Integer.valueOf()` 对 **-128 ~ 127** 的值返回缓存对象：

```java
Integer a = 127, b = 127;
a == b;   // true（同一缓存对象）

Integer c = 128, d = 128;
c == d;   // false（新建对象，引用不同）
```

**结论**：Integer 比较值相等必须用 `.equals()`，禁止用 `==`。

### 自动拆箱的 NPE 陷阱

```java
Integer i = null;
int n = i;     // NullPointerException！（i.intValue() 对 null 调用）

// 危险场景：三目运算符
Boolean flag = null;
int k = flag ? 1 : 0;  // 自动拆箱 → NPE
```

### RPC 接口：应该用基本类型还是包装类？

**返回值用包装类**：区分"值为 0"和"没有值（null）"，基本类型无法表达 null 语义，调用方可能误判。

**参数：看业务**：必传参数用基本类型（防 null 传入）；可选参数用包装类。

### 金额计算：为什么不能用 double/float？

浮点数（IEEE 754）以二进制分数存储，`0.1` 无法精确表示：

```java
0.1 + 0.2 == 0.30000000000000004  // 不等于 0.3
```

**金额用 `BigDecimal`**（或整型分/厘存储）：
- 用 `BigDecimal(String)` 而非 `BigDecimal(double)` 构造（后者仍有精度问题）
- `equals()` 比较值和精度（`1.0 ≠ 1.00`）；值相等用 `compareTo() == 0`

## 关键权衡

1. **装箱有开销**：频繁装拆箱产生大量临时对象，触发 GC；循环中避免对包装类做数学运算
2. **空指针风险**：包装类可为 null，拆箱时未判空会 NPE；POJO 字段用包装类，方法内计算用基本类型
3. **==比较陷阱**：包装类缓存范围外的 `==` 返回 false，统一用 `.equals()`

## 与其他概念的关系

- 被 [[机制-泛型类型擦除]] 约束：泛型集合不能存基本类型 → 必须用包装类
- 与 [[机制-String不可变性]] 相似：同为不可变类 + 有缓存机制（Integer Cache vs String Pool）
- 在 L4 集合框架：`HashMap<Integer, List<String>>` 中 Integer key 的 hashCode 稳定（不可变）

## 应用边界

**用包装类**：POJO 字段（区分 null）；泛型集合元素；RPC 接口返回值；可选参数。

**用基本类型**：局部变量和方法内计算（性能优先）；不需要 null 语义的必填参数。

**金额**：`BigDecimal` 配合 `String` 构造，`compareTo` 比较。
