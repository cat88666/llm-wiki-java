# 算法高频题型

> 大厂 Java 后端面试的算法重点：**理解题目背后的数据结构选型**，而非死记模板。核心是 HashMap/堆/双指针/DP 这几种范式的组合应用。

---

## 时间复杂度速查

| 复杂度 | 名称 | 典型场景 | 面试参考 |
|--------|------|---------|---------|
| O(1) | 常数 | 哈希表查找 | 设计题目标 |
| O(log n) | 对数 | 二分查找、平衡树 | n=10^6 约20次操作 |
| O(n) | 线性 | 一次遍历 | 最常见最优解 |
| O(n log n) | 线性对数 | 归并/快速排序 | 面试排序基准 |
| O(n²) | 平方 | 暴力两层循环 | 有 n=10^5 时不可接受 |

## 高频题型与核心思路

### LRU 缓存（O(1) get/put）

**要求**：`get(key)` 和 `put(key, value)` 均为 O(1)，超容量时淘汰最久未使用的 key。

**数据结构选型**：HashMap（O(1) 查找）+ 双向链表（O(1) 移动节点到头部 / 删除尾部）。

```
HashMap: key → Node
双向链表: head ←→ [最近用] ←→ ... ←→ [最久未用] ←→ tail

get(key):  找到 Node → 移到 head
put(key):  存在→更新+移到head；不存在→新建放head，若超容则删tail
```

**Java 捷径**：`LinkedHashMap(capacity, 0.75f, true)` 设 `accessOrder=true`，重写 `removeEldestEntry` 即可。

**完整实现（手写版，面试标准答案）**：

```java
class LRUCache {
    private Map<Integer, Node> map = new HashMap<>();
    private Node head = new Node(), tail = new Node();
    private int capacity;

    public LRUCache(int capacity) {
        this.capacity = capacity;
        head.next = tail; tail.prev = head;
    }

    public int get(int key) {
        Node node = map.get(key);
        if (node == null) return -1;
        moveToHead(node);
        return node.value;
    }

    public void put(int key, int value) {
        Node node = map.get(key);
        if (node != null) { node.value = value; moveToHead(node); }
        else {
            node = new Node(key, value);
            map.put(key, node); addToHead(node);
            if (map.size() > capacity) { map.remove(tail.prev.key); removeNode(tail.prev); }
        }
    }
    // addToHead / removeNode / moveToHead 操作双向链表
}
```

**线程安全怎么做**：①全方法 `synchronized`：简单但读写互斥，高并发差；②读写锁：`get` 也需要 `moveToHead`（写链表），退化成写锁，与①无区别；③**推荐：直接用 Caffeine**（ConcurrentHashMap + Ring Buffer 异步维护淘汰顺序，近似 LRU，性能高一个数量级）。生产场景不要自己实现并发 LRU。

### Top K 问题

| 方案 | 复杂度 | 适用 |
|------|--------|------|
| **小顶堆（维护 K 个）** | O(n log k) | n 很大，k 较小，在线流式 |
| **快速选择（partition）** | 平均 O(n)，最坏 O(n²) | 一次性，不需要排序 |
| 全排序 | O(n log n) | n 较小 |

**小顶堆解法**：遍历数组，堆大小 < k 则直接加入；否则若当前元素 > 堆顶，替换堆顶并重新堆化。最终堆中就是 Top K 大的元素（堆顶是第 K 大）。

```java
PriorityQueue<Integer> minHeap = new PriorityQueue<>(k);
for (int num : nums) {
    if (minHeap.size() < k) minHeap.offer(num);
    else if (num > minHeap.peek()) { minHeap.poll(); minHeap.offer(num); }
}
```

### 海量数据处理

**经典模式**：哈希分片 → 局部处理 → 合并结果

| 问题 | 解法 |
|------|------|
| **100亿整数，找出现次数最多的前100** | 哈希分片到 1000 个文件 → 每文件 HashMap 统计 → 小顶堆合并各文件 Top 100 |
| **两个各100亿行 URL 文件，找共同 URL** | 同一 URL 按 `hash(url) % N` 分到相同编号文件 → 每对同编号文件求交集（HashSet）|
| **10亿数据去重** | 布隆过滤器（允许少量误判）/ BitMap（纯整数场景）|
| **实时 UV 统计** | HyperLogLog（12KB 内存，误差 0.81%）|

**关键原则**：单机内存装不下时，先哈希分片让相同 key 归到同一文件，再单文件处理。

### 排序算法选型

| 算法 | 平均 | 最坏 | 空间 | 稳定 | 特点 |
|------|------|------|------|------|------|
| 归并排序 | O(n log n) | O(n log n) | O(n) | ✓ | 链表排序首选 |
| 快速排序 | O(n log n) | O(n²) | O(log n) | ✗ | 最常用，pivot 随机化避免最坏 |
| 堆排序 | O(n log n) | O(n log n) | O(1) | ✗ | 原地，但缓存不友好 |
| TimSort | O(n log n) | O(n log n) | O(n) | ✓ | Java `Arrays.sort(Object[])` 实现 |

**Java 实现细节**：基本类型用双轴快速排序（DualPivotQuicksort）；对象类型用 TimSort（归并变体，稳定）。

**快速排序优化**：① 随机选 pivot（避免有序数组退化为 O(n²)）；② 三路划分（大量重复元素时 O(n)）。

### 二分查找

**标准模板**（`left <= right`，每次确定性排除）：

```java
int left = 0, right = nums.length - 1;
while (left <= right) {
    int mid = left + (right - left) / 2;  // 防 int 溢出
    if (nums[mid] == target) return mid;
    else if (nums[mid] < target) left = mid + 1;
    else right = mid - 1;
}
return -1;
```

**变体**：
- 找第一个 ≥ target（lower_bound）：`if (nums[mid] >= target) right = mid - 1; else left = mid + 1;`，返回 `left`
- 旋转有序数组：先判断哪半段有序，再确定 target 在哪半段

### 双指针 & 滑动窗口

**对撞指针（有序数组/两端收缩）**：

```
两数之和（有序版）: left++（和太小）/ right--（和太大）
接雨水: min(height[left], height[right]) 决定移哪边
三数之和: 固定一个，内层对撞指针
```

**快慢指针（链表）**：
- 判环：快指针2步/慢指针1步，相遇则有环
- 找环入口：相遇后一个指针回头节点，同速前进，再次相遇即入口
- 找中点：快指针到尾时，慢指针在中间

**滑动窗口（维护窗口内的状态）**：

```java
// 最长无重复子串
Map<Character, Integer> map = new HashMap<>();
int left = 0, maxLen = 0;
for (int right = 0; right < s.length(); right++) {
    char c = s.charAt(right);
    if (map.containsKey(c)) left = Math.max(left, map.get(c) + 1);
    map.put(c, right);
    maxLen = Math.max(maxLen, right - left + 1);
}
```

### 动态规划

**解题四步**：定义状态 → 写转移方程 → 确定初始值 → 确定计算顺序

**经典题型转移方程**：

| 题目 | 状态定义 | 转移方程 |
|------|----------|---------|
| 最大子数组和（Kadane）| `dp[i]` = 以 `nums[i]` 结尾的最大子数组和 | `dp[i] = max(nums[i], dp[i-1] + nums[i])` |
| 最长递增子序列（LIS）| `dp[i]` = 以 `nums[i]` 结尾的 LIS 长度 | `dp[i] = max(dp[j]+1)，j<i 且 nums[j]<nums[i]` |
| 最长公共子序列（LCS）| `dp[i][j]` = s1 前 i 字符与 s2 前 j 字符的 LCS | 相等：`dp[i-1][j-1]+1`；不等：`max(dp[i-1][j], dp[i][j-1])` |
| 0-1 背包 | `dp[j]` = 容量 j 时的最大价值 | `dp[j] = max(dp[j], dp[j-w]+v)`（逆序遍历）|
| 爬楼梯/斐波那契 | `dp[i]` = 到第 i 阶的方案数 | `dp[i] = dp[i-1] + dp[i-2]` |

**空间优化**：当 `dp[i]` 只依赖 `dp[i-1]`，可滚动数组压缩为 O(1) 空间。

### 树（二叉树高频）

**遍历方式选择**：
- 前序（根左右）：序列化/打印树结构
- 中序（左根右）：BST 输出有序序列
- 后序（左右根）：计算目录大小、删除节点（先删子）
- 层序（BFS）：按层处理、求树高度

**高频题思路**：
- **最近公共祖先（LCA）**：后序遍历，在左右子树都找到目标节点时当前节点即为 LCA
- **路径和**：DFS 携带当前路径和，到叶节点时判断是否等于 target
- **对称二叉树**：递归比较左子树的左 vs 右子树的右、左子树的右 vs 右子树的左

### 图算法（工程高频）

**并查集（Union-Find）**：快速判断两节点是否连通，路径压缩后近似 O(1)：

```java
int find(int x) {
    if (parent[x] != x) parent[x] = find(parent[x]); // 路径压缩
    return parent[x];
}
void union(int x, int y) {
    int px = find(x), py = find(y);
    if (px != py) parent[px] = py;
}
```

**应用**：朋友圈连通分量数、冗余连接（检测环）、网络延迟（Kruskal 最小生成树）。

**拓扑排序（Kahn 算法）**：任务依赖/课程依赖，检测有向图是否有环：

```
1. 统计所有节点入度
2. 入度为 0 的节点入队
3. 弹出队头节点，其邻居入度 -1，新的入度为 0 的节点入队
4. 若最终处理的节点数 < 总节点数，说明有环
```

## 规律总结

1. **O(1) 操作 → 选 HashMap / 堆**：要求 O(1) 查找必然用 HashMap；要求 O(1) 取最值必然用堆
2. **有序序列 → 先考虑二分**：在有序范围内查找/判断，优先二分而非线性扫描
3. **子数组/子串 → 双指针/滑窗**：连续区间的最优解通常是滑动窗口（左右指针各移动一次，O(n)）
4. **最优子结构 → 动态规划**：问题可以分解为子问题且子问题有重叠时用 DP；最优子结构不明显时用回溯
5. **大数据 → 哈希分片 + 局部处理**：数据量超内存就分片，相同 key 的分到同一片

## 与其他概念的关系

- **依赖 [[机制-优先队列]]**：Top K / 合并 K 个有序链表 / 数据流中位数都依赖堆的 O(log n) 操作
- **依赖 [[机制-HashMap]]**：LRU、两数之和等大量题目以 HashMap 为核心
- **依赖 [[概念-图]]**：并查集 / 拓扑排序 / BFS最短路径是图算法在工程中的三大应用
- **关联 [[概念-布隆过滤器]]**：海量数据去重/判存在场景，布隆过滤器是 BitMap 的概率性升级
