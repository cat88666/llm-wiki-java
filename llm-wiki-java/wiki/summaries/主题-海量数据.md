# 海量数据处理

> 海量数据问题的本质是**内存不足以容纳全量数据**，解题思路只有两条：① 压缩表示（BitMap/布隆过滤器）；② 分治处理（哈希分片 + 分块 + 归并）。

## 一、选型速查

| 场景 | 推荐方案 | 关键约束 |
|------|----------|----------|
| 整数集合去重/判存在 | **BitMap** | 整数范围已知且不太大 |
| 海量字符串判存在（允许误判）| **布隆过滤器** | 不可删除；假阳性不可避免 |
| 海量数据排序（内存不足）| **外部归并排序** | 分块→块内排序→多路归并 |
| 海量日志 TopK | **哈希分片 + 堆** | 先分而治之，再全局汇总 |
| 关键词/黑名单匹配 | **Trie 树 / AC 自动机** | 多模式串匹配 |

## 二、BitMap：整数去重与判存在

### 40亿QQ号限制1G内存去重

40亿个 int（4字节）直接存需要 `4 × 40亿 = 14.9G`。

**BitMap 方案**：每个数字占 1 bit，10位数字上限 9,999,999,999：
```
10,000,000,000 bit ÷ 8 ÷ 1024 ÷ 1024 = 1192 MB ≈ 1.16G
```
> 1G 内存略不够，可以选择：①把范围分成2段，两次扫描；②用 EWAHCompressedBitmap 压缩稀疏位图。

**核心操作**：将 QQ 号 `N` 标记存在 → `bitmap[N/8] |= (1 << (N%8))`。最终遍历所有为 1 的位，即为去重结果。

### 1亿整数（范围1~2亿）是否存在：100M内存

数字范围 2亿，BitMap 需要：`2亿 bit ÷ 8 ÷ 1024 ÷ 1024 = 25 MB`，远小于 100M，直接用 BitMap。

## 三、布隆过滤器：内存估算

### 公式

```
位数组大小 m = -n × ln(p) / (ln2)²
```
- `n`：元素数量
- `p`：期望误判率
- `(ln2)² ≈ 0.4805`

### 5亿数据的内存估算

| 误判率 p | 计算结果 | 内存占用 |
|---------|---------|---------|
| 0.1% (0.001) | m ≈ 71.87亿 bit | **约 856 MB** |
| 1% (0.01) | m ≈ 47.92亿 bit | **约 571 MB** |

哈希函数个数：`k = m/n × ln2`

**布隆过滤器 vs BitMap**：
- BitMap 要求整数范围已知且连续；布隆过滤器适用任意类型数据（字符串、URL）
- 布隆过滤器有假阳性（误判存在但实际不存在）；BitMap 无误差
- **缓存穿透防护**首选布隆过滤器；**整数去重**首选 BitMap

## 四、外部归并排序：1TB数据32G内存

### 核心思路

```
阶段1：分块排序（内存能装多少装多少）
  1TB ÷ 30G/块 ≈ 36个块（预留2G给OS和排序操作）
  每块读入内存 → 快排/Timsort → 写回磁盘有序临时文件
  → 得到36个局部有序文件

阶段2：多路归并（小顶堆）
  同时打开 K 个文件（K 路归并），为每路分配输入缓冲区
  每路取堆顶最小元素 → 写输出缓冲区 → 缓冲区满则刷盘
  K 的选择：K = floor((可用内存 - 输出缓冲) / 每路输入缓冲)
  实践：取 8~18 路（平衡归并次数和内存利用率）

  36个文件 → 9路归并 → 4个文件 → 4路归并 → 1个结果文件
```

**为什么用小顶堆**：K 路同时归并，每次从 K 个堆顶取最小值，O(log K) 时间，无需比较所有元素。

## 五、海量日志 TopK：哈希分片 + 堆

### 1TB搜索日志找搜索量最高10个关键词

```
步骤1：哈希分片
  将 1TB 日志按 hash(keyword) % 100 分成 100 个文件
  同一个关键词必然在同一个文件中（约 10G/文件）

步骤2：各文件内统计
  逐个文件加载内存，用 HashMap 统计词频
  取本文件内 Top10（小顶堆）

步骤3：全局汇总
  100 个文件各自的 Top10 → 1000 条记录
  全部加载内存，再次堆排序取全局 Top10
```

**变体：单机内存只有2G但数据是TB级**：同样先哈希分片，保证每片能装入内存。

## 六、字符串匹配：Trie 树与 AC 自动机

### 敏感词过滤

**Trie树（前缀树）**：
```
敏感词库 → 构建 Trie
  "暴力" → 根 → '暴' → '力'(end)
  "暴徒" → 根 → '暴' → '徒'(end)
查询时沿树遍历，到 end 标记即命中
```
- 构建一次，O(m) 查询（m为敏感词总字符数）
- 问题：只能从固定位置开始匹配，正文中间出现的敏感词需要滑动

**AC自动机（Aho-Corasick）**：Trie + 失败指针（KMP 思想扩展）
- 单次扫描文本，O(n+m) 时间，同时匹配多个模式串
- **生产推荐**：AC自动机，或 ES 全文检索（自动分词）

### 黑名单URL过滤

| 方案 | 内存 | 查询速度 | 特点 |
|------|------|----------|------|
| HashSet | 高（存完整URL）| O(1) | 精确，内存大 |
| **布隆过滤器** | 极低 | O(k) k=哈希函数数 | 允许误判，适合亿级黑名单 |
| Trie + 域名分层 | 中 | O(域名层数) | 适合域名前缀匹配 |

**骚扰电话号码存储**：数量级在亿以内时，BitMap（手机号是11位数字，范围确定）；超过范围时用布隆过滤器。
