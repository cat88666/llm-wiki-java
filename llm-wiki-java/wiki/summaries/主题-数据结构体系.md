# 主题：数据结构体系

> L4 数据结构层。面向 Java 工程师的核心数据结构知识聚合，是集合框架（L4 容器）、存储层索引（L5 MySQL/Redis）、算法面试的共同地基。

## 核心高频考点

### 1. 红黑树（高频 ⭐⭐⭐）

- **5 条规则**：黑根；红节点子必黑；所有路径黑高相同 → 最长路径 ≤ 最短 × 2
- **Java 使用**：`HashMap`（链表长 ≥ 8 → 红黑树）；`TreeMap`/`TreeSet`
- **vs AVL**：红黑树旋转次数少，写多读少优选；AVL 查询略快，读多写少优选

### 2. B+ 树（高频 ⭐⭐⭐）

- **数据只在叶节点**：非叶节点纯路由，单页存更多键，树更矮（通常 3-4 层）
- **叶节点双向链表**：天然支持范围查询、顺序扫描
- **MySQL InnoDB 选择原因**：低树高（少 IO）+ 链表（范围查询）
- **B 树 vs B+ 树**：B 树单点查询可提前终止；B+ 树范围查询更优

### 3. 堆 / Top K（高频 ⭐⭐⭐）

- **反直觉口诀**：找最大 K → 用**小**顶堆（维护 K 个候选，新元素 > 堆顶才替换）
- **原因**：小顶堆只存 K 个元素，内存 O(k)；大顶堆需存全量数据
- **Java**：`PriorityQueue`（默认小顶堆）；大顶堆 `new PriorityQueue<>(Comparator.reverseOrder())`
- **TP99**：大顶堆 + 小顶堆维护 99:1 分割点，实时获取

### 4. 线性结构（高频 ⭐⭐）

- **数组 vs 链表**：数组随机访问 O(1)，插中间 O(n)；链表插删 O(1)（已有引用），访问 O(n)
- **栈不要用 `java.util.Stack`**：继承 Vector，所有方法 synchronized，用 `ArrayDeque`
- **循环队列**：ArrayDeque（循环数组）比 LinkedList 队列性能更好（Cache 友好）

### 5. 图的遍历（中频 ⭐⭐）

- **BFS**：用队列，按层扩散，无权图最短路径
- **DFS**：用栈/递归，一路到底再回溯，适合连通性、环检测
- **Dijkstra**：有权最短路，BFS 基础 + 小顶堆选最短距离节点

### 6. BitMap 与布隆过滤器（中频 ⭐⭐）

- **BitMap**：1 bit = 1 个整数的存在状态，10 亿整数仅需 125 MB
- **布隆过滤器**：k 个哈希 → k 个 bit 位；有误判（假阳性），无漏判（假阴性）
- **缓存穿透防护**：布隆过滤器前置拦截不存在的 Key，保护数据库

## 结构间依赖关系

```
线性结构（栈/队列）
    ↓ 是 DFS/BFS 的运行时容器
图遍历（DFS/BFS）
    ↓ BFS + 小顶堆
Dijkstra 最短路径
         ↑ 依赖
堆（优先队列）
    ↓ 数组实现堆
数组（线性结构）

红黑树
    ↓ 用于磁盘时退化为
B+ 树（多路化 + 降低 IO）
    ↓ 是
MySQL InnoDB 索引底层

BitMap
    ↓ 扩展为
布隆过滤器（多哈希 + 多 bit 位）
    ↓ 应用于
缓存穿透防护 / URL 去重
```

## L4 与上下层的关系

| 上层依赖 | 具体联系 |
|---------|---------|
| L4 容器（集合框架）| `HashMap` 内桶用红黑树；`PriorityQueue` 是堆；`ArrayDeque` 是循环数组 |
| L5 MySQL | InnoDB 索引 = B+ 树；回表 = 二级索引叶节点存主键，再查聚簇索引 |
| L5 Redis | Sorted Set（ZSet）底层 = 跳表（有序链表变体）；BitMap 命令直接暴露位图 |
| L3 并发 | `PriorityBlockingQueue` 是线程安全堆；`ConcurrentSkipListMap` 是跳表 |
