## 一、暖场 & 自我介绍

### 自我介绍

12 年 Java 研发，前期游戏+区块链高并发，后转型支付金融，目前在东京。两段核心经历：腾娱把游戏平台 JVM Full GC 调到 0；支付界做 Seata TCC，跨行转账异常链路资金损失率降到零。

## 二、行为 & 管理 & 反问

### 技术决策分歧怎么处理？

网关重构：一半人 SCG，一半人 Netty。让两派各写 1 天 PoC 压测说话。Netty 延迟低 30% 但维护成本 3 倍。决策：游戏内网关 Netty（性能第一），运营后台 SCG（迭代第一）。数据驱动，不拍板。

### 怎么培养初级开发？

渐进式任务（配置改→接口→模块→系统设计）+ Code Review 解释"为什么"建立工程直觉 + 参与线上问题排查（旁边兜底）+ 明确半年目标定期 1:1。核心：给方向、给机会、给反馈，不替他做决定。

### 印象最深的线上事故？

支付某银行渠道回调超时激增，2000 笔订单卡在"支付中"：
1. **止血（5min）**：回滚该渠道到上一版本
2. **定位（10min）**：HTTP Client 连接池从 200 改成了 20，高峰期耗尽
3. **修复**：修正参数，5% 灰度验证后全量
4. **补单**：向银行查单，批量更新 2000 笔
5. **复盘**：CI 增加关键配置参数变更的额外审批

15 分钟恢复，零资损，30 分钟补单完毕。
