# 德州扑克核心算法

> 一套支持标准德州/短牌/奥马哈三变体的游戏引擎，核心由四层组成：**牌型评估**（手牌比较）→ **胜率/Outs计算**（实时概率）→ **智能保险**（赔率方程求解）→ **底池结算**（边池+抽水分配）。每层均有独立的数学模型，合计主要算法代码 ~3000 行。

## 一、牌型评估系统

### 数据结构

```java
// PokerCard：基础牌型表示
// 花色编码：Spades=3, Hearts=2, Clubs=1, Diamonds=0
// 牌值编码：2-10 取本身，J=11, Q=12, K=13, A=14
// 短牌中 A 可视为 14 或 1（轮盘顺子特例处理）
```

### 工厂模式：按变体分配评估器

```
PokerHandEvaluatorFactory.getEvaluator(gameType)
  ├─ isTexas(type)     → NormalPokerHandEvaluator（标准德州）
  ├─ isShortDeck(type) → ShotPokerHandEvaluator（短牌）
  └─ isOmaha(type)     → AmhNormalPokerHandEvaluator（奥马哈）
```

### 牌型层级（标准德州，从高到低）

| 等级 | 牌型 | 判断逻辑 |
|------|------|---------|
| 10 | 皇家同花顺 | 同花顺且最高为A |
| 9 | 同花顺 | 连续5张同花，A可作1（轮盘顺） |
| 8 | 四条 | 4张相同点数 |
| 7 | 葫芦 | 三条 + 对子 |
| 6 | 同花 | 5张同花，不连续 |
| 5 | 顺子 | 连续5张不同花 |
| 4 | 三条 | 3张相同点数 |
| 3 | 两对 | 两个不同的对子 |
| 2 | 一对 | 一个对子 |
| 1 | 高牌 | 无任何组合 |

> **短牌差异**：葫芦 > 同花（因移除2-5后同花更难凑齐），`CardComparator` 在短牌模式下调整了比较顺序。

### 核心比较算法：多项式系数编码

**设计目的**：避免每次都重建牌组排列，用一个整数值直接表示牌型强弱，O(1) 比较。

```
同花（5张高牌编码）：
  result = c1×20⁴ + c2×20³ + c3×20² + c4×20 + c5
  （c1~c5 为从高到低排序的5张同花牌点数）

两对：result = pair1×400 + pair2×20 + kicker
一对：result = pair×8000 + k1×400 + k2×20 + k3
三条：result = three×400 + k1×20 + k2
四条：result = four×100 + kicker
葫芦：result = three×100 + pair
```

所有牌型最终都映射到一个可直接 `>` 比较的整数，避免多字段比较的复杂性。

### 奥马哈的特殊处理（必须用恰好2张手牌+3张公牌）

```
1. 对4张手牌生成 C(4,2) = 6 种组合
2. 对5张公牌生成 C(5,3) = 10 种组合
3. 枚举所有 6×10 = 60 种五张牌组合
4. 选取评估分最高的一种作为最终成牌
```

## 二、胜率与 Outs 计算

### 分阶段策略

| 发牌阶段 | 公牌数 | 计算方式 | 原因 |
|---------|--------|---------|------|
| Pre-Flop | 0 | 查表法（预建2万+组合） | 组合固定，提前缓存 |
| Flop | 3 | 完整枚举剩余牌 | C(47,2)=1081 种，可接受 |
| Turn | 4 | 蒙特卡洛模拟 | 精度要求较低 |
| River | 5 | 精确比牌（公牌已固定） | O(n)，最快 |

### Outs 计算核心逻辑（findTwoHandCardOuts）

**Outs 定义**：落后玩家翻出后能反超领先玩家的那些牌。

```
输入：
  - baseHand（领先玩家手牌）
  - hand2（落后玩家手牌）
  - unShowCards（剩余未发牌）

逻辑分支：
  1. baseHand 有对子且更高 → 返回 hand2 能凑出更高对子的outs
  2. hand2 有对子且更高 → 返回 baseHand 能凑出更高对子的outs
  3. 都是高牌 → 返回 hand2 中比 baseHand 最高牌更大的所有点数
```

**Outs 类型**：
- **反超 Outs（Turn-Tide）**：落后者能反超的牌，`outsType=1`
- **平分 Outs（Divide）**：多人同时获益导致平分，`outsType=2`

### 四种投保方案

| 方案 | 类型常量 | 投保逻辑 |
|------|---------|---------|
| 方案0 | `WAITING_PROFIT` | **等利**：投保后爆牌与不爆牌收益相同 |
| 方案1 | `ONE_FOURTH_POT` | 期望值使1/4底池损失平摊 |
| 方案2 | `ONE_EIGHTH_POT` | 期望值使1/8底池损失平摊 |
| 方案3 | `ONE_SIXTEENTH_POT` | 期望值使1/16底池损失平摊 |

### 方案0：等利方案推导

```
目标：投保后无论 Outs 中还是没中，收益相同

设 T = 总投保额，ri = 第 i 个 Outs 的赔率
  投保后爆牌收益 = 新控制金额 + T×ri - T = 新控制金额 + T(ri-1)
  不投保收益     = 原控制金额

令两者相等：
  T = (原控制金额 - 新控制金额) / (ri + 1)
```

### 方案1-3：期望值方程组（核心数学）

**问题**：多个 Outs 的赔率不同，如何在总投保额 T 的约束下，使每个 Outs 中奖的期望收益等于目标值 Hi？

**方程组建立**：
```
每个 Outs i 的投保额 Ai 满足：
  (ri + 1) × Ai = Hi + T       // 赔付额 = 目标收益 + 总投入
  Σ Ai = T                     // 所有Outs投保额之和等于总投保额
```

**求解 T（迭代法）**：
```
step 1: 将 Ai = (Hi + T) / (ri + 1) 代入 ΣAi = T
        → T = Σ[(Hi + T) / (ri + 1)]
        → T × [1 - Σ(1/(ri+1))] = Σ[Hi/(ri+1)]

step 2: T = [Σ Hi/(ri+1)] / [1 - Σ 1/(ri+1)]

step 3: 由 T 反推各 Outs 的投保额 Ai = (Hi + T) / (ri + 1)

step 4: 若存在 Ai < 0（不合理），将负数清零并重新计算，
        最多迭代 20 次直到收敛
```

### 方案可用性检查

```
条件1: 赢利额 >= 期望收益值（不能亏钱）
条件2: 单池投保总额 ≤ 上限
       Turn 圈：上限 = 自己投入 × 0.25
       River 圈：上限 = 控制金额 - 上轮投保额
条件3: 控制金额 ≥ 10 × 大盲（底池充足）
条件4: 方案投保额 > 0
```

### 背回（续投）逻辑

```
条件：上一轮在该池子投保，且上一轮所有 Outs 均未中
→ 需要背回（继续本轮投保保持风险对冲）

背回投保额 = 上轮总投保额 / 本轮赔率
（若大盲≥1，向上取整；否则保留2位小数向上取整）
```

## 四、底池结算系统

### 边池生成逻辑

多人 all-in 时，按各人 all-in 金额切分底池：

```
示例：A all-in 投 100，B 投 200，C 投 200

主池 = min(100,200,200) × 3 = 300    参与：A, B, C
边池1 = (200-100) × 2 = 200          参与：B, C

若 B 在 150 时 all-in（B投入合计150）：
  主池 = 100 × 3 = 300               参与：A, B, C
  边池1 = 50 × 2 = 100               参与：B, C
  边池2 = 50 × 1 = 50                参与：C（超出B all-in的部分）
```

### 比牌赢家确定（calculateMaxComparison）

```
只有1人未弃牌       → 该玩家赢
只有1人最终亮牌     → 亮牌玩家赢
多人亮牌            → 按牌型编码大小比较
                      i.  先按牌型等级（1-10）
                      ii. 同牌型按多项式编码数值
同牌型同编码（平局） → 并列赢家，平分底池
```

**并列平分余数处理**：
```
per_player = totalPot / winners.size()
remainder  = totalPot % winners.size()
→ 余数分给位置最靠前（最先行动）的赢家
```

### 抽水算法

两种模式：

**模式A：按底池比例抽（POT_SCALE）**
```
应抽总额 = 总有效底池 × 抽水比例%
实际总抽 = min(应抽总额, 抽水上限)   // 上限 = 大盲倍数 × 大盲注

单池应抽 = 单池金额 × 实际总抽 / 总金额
尾差（小数误差） → 归入最大底池
```

**模式B：按盈利比例抽（PROFIT_SCALE）**
```
单池盈利   = 底池金额 - 赢家在该池投入
总盈利     = Σ单池盈利
应抽总额   = 总盈利 × 抽水比例%
实际总抽   = min(应抽总额, 抽水上限)

单池应抽   = 单池盈利 × 实际总抽 / 总盈利
尾差       → 归入最大盈利底池
```

**玩家最终收益公式**：
```
玩家获得的抽水贡献 = 玩家该池投入 × 该池抽水 / 该池总投入
最终净收益 = 赢得金额(已扣水) - 玩家投入 + 抽水贡献
```

### 三种保险模式下的结算差异

| 模式 | 常量 | 盈亏计算 | 特殊处理 |
|------|------|---------|---------|
| 经典保险 | `CLASSIC_INSURANCE=1` | `Σ(补偿金 - 投保额)` | 亏损不超过该池赢得金额 |
| EV保险 | `EV_INSURANCE=2` | `EV补偿 - 玩家实际赢得` | 赢家不参与抽水 |
| 智能保险 | `SMART_INSURANCE=3` | `Σ(补偿金 - 投保额)`（跨池汇总） | 总亏损不超过游戏盈亏 |

## 六、性能关键点

1. **多项式编码**：牌型比较 O(1)，无需排序或多字段比较
2. **分阶段策略**：Pre-Flop 查表，River 精确比牌，中间阶段枚举/模拟
3. **保险方案迭代收敛**：最多 20 次迭代，负值清零后重算，避免数值不稳定
4. **奥马哈 60 种组合**：C(4,2)×C(5,3) 固定数量，无需动态优化
5. **边池尾差归一**：统一归入最大底池，避免分配后筹码不守恒

## 七、面试回答要点

### 牌型比较如何快速？
> "使用多项式系数编码，把5张牌的强弱映射到一个整数。同花就是 c1×20⁴+c2×20³+...+c5，两对就是 pair1×400+pair2×20+kicker。这样比较两手牌只需一次整数大小比较，O(1)。"

### 保险赔率怎么算？
> "智能保险支持同时对多张 Outs 投保。核心是建立方程组：每个 Outs 的投保额 Ai = (目标收益Hi + 总投保额T) / (赔率ri + 1)，且 ΣAi = T。消元后得到 T 的解析式，再反推各 Ai。若有负值则迭代修正，最多 20 次收敛。"

### 边池如何分配？
> "多人 all-in 时按各人 all-in 金额切分主池和多个边池。每个边池只有当时还有筹码参与的玩家才能竞争。结算时从主池开始逐级比牌，赢家拿走该池，不同边池可能是不同赢家。余数（小数尾差）归入位置最前的赢家或最大底池。"

### 如何支持多变体？
> "工厂模式：PokerHandEvaluatorFactory 根据 gameType 分配不同评估器；BaseCalculator 抽象基类封装公共逻辑，ShortCalculator/NormalCalculator/AmhNormalCalculator 分别覆盖差异点（如短牌葫芦>同花、奥马哈的2+3强制选牌逻辑）。新增变体只需扩展工厂和新增 Calculator 实现。"
